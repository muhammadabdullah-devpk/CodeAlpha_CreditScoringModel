CreditAI Architecture & Methodology

Learn how CreditAI pre-processes, models, calibrates and scores financial portfolios.

1. Dataset & Generation

The model is trained on a synthetic representation of institutional consumer borrowing records containing 8,000 samples. The dataset includes numerical attributes (Income, Savings, Age, Debt, Prior Defaults, Utilization, etc.) and categorical tags (Education, Homeownership type, Employment type, Loan purpose).

2. Feature Engineering & Pre-processing

We construct four major derived ratio fields to reflect borrowing risk metrics:

Debt to Income Ratio (DTI) DTI = (Monthly Debt * 12) / Annual Income
Loan to Income Ratio (LTI) LTI = Loan Amount / Annual Income
Payment Burden Ratio Burden = (Loan Amount / Term) / (Annual Income / 12)
Stability Score Reflects combined Job Tenure, Credit History Length, and employment stability flags.

A scikit-learn ColumnTransformer handles missing data via median/mode imputers and scales continuous parameters using a StandardScaler, while categorical fields are one-hot encoded.

3. Classification Engine

An ensemble RandomForestClassifier comprising 200 trees is fitted to the pre-processed input matrices. We apply balanced subsample weighting to handle positive/negative class frequency imbalances.

4. Threshold Calibration

Instead of standard default classification boundaries (0.5), we scan candidate decision thresholds over the validation dataset to optimize a blended metric function:

Blended Score = 0.55 * F1 + 0.25 * Recall + 0.20 * Precision

This strategy minimizes false approvals while preserving standard acceptance throughput.
